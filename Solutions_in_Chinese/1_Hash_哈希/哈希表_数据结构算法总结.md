# 一、什么是 Hash（哈希）

## 本质

Hash：

```text
通过 key
快速找到 value
```

例如：

```python
student = {
    "Tom": 95,
    "Jack": 88
}
```

查询：

```python
student["Tom"]
```

速度非常快。

---

# 为什么快？

因为底层：

```text
Hash Table（哈希表）
```

通过：

```text
Hash Function（哈希函数）
```

把 key 转换成数组下标。

因此查询平均时间复杂度：

$$
O(1)
$$

# 四、set（LeetCode 高频）

## 本质

```text
不重复元素集合，1. 去重（最核心），2. 无序（重点）。
set 本质是：
Hash Table
不是数组。
所以：
不保证顺序
不保证排序
输出顺序可能变化
```

底层也是 Hash Table。

---

# 创建

```python

```

---

# 去重

```python

set(nums)
```

结果：

```python

```

---

# 添加元素

```python

```

---

# 删除元素


---

# 查询

```python

```

# 六、defaultdict（面试高频）

```python
from collections import defaultdict
```

---

# 本质

```text

自动创建默认值
```

---

# 普通 dict 问题

```python

```

报错原因：

```text

```

---

```python
d = defaultdict(list)
```

现在：

```python

```

自动变成：

```python

    "a":[1]
}
```

# 十、Hash 的底层计算机知识（真正核心）

---

# 1. Hash Function（哈希函数）

作用：

```text
把 key 转换成整数下标
```

例如：

```python
hash("abc")
```

面试一句话回答：

```text
哈希函数的目标是让数据尽可能均匀分布，从而实现 O(1) 查询。
```

---

# 2. Hash Table（哈希表）

本质：

```text
数组 + 哈希函数
```

核心思想：

```text
通过计算下标实现快速访问
```

面试一句话回答：

```text
哈希表通过 key 直接定位数组位置，因此查询速度非常快。
```

---

# 3. Hash Collision（哈希冲突）

问题：

```text
不同 key
可能映射到同一个位置
```

例如：

```text
A -> 5
B -> 5
```

面试一句话回答：

```text
哈希冲突无法完全避免，只能通过优秀哈希函数降低概率。
```

---

# 4. Chaining（拉链法）

解决冲突的方法之一。

思想：

```text
同一个位置挂链表
```

面试一句话回答：

```text
拉链法通过链表存储冲突元素，是最经典的冲突解决方案。
```

---

# 5. Open Addressing（开放寻址）

冲突时：

```text
寻找下一个空位置
```

面试一句话回答：

```text
开放寻址不使用链表，而是在数组中继续寻找空位。
```

---

# 6. 为什么 Hash 平均是 O(1)

因为：

```text
数据会均匀分布
```

大部分情况下：

```text
一次即可找到
```

面试一句话回答：

```text
均匀分布使得哈希表平均只需一次访问即可完成查询。
```

---

# 7. Hash 为什么最坏是 O(n)

极端情况下：

```text
所有元素冲突
```

会退化成链表。

面试一句话回答：

```text
如果大量哈希冲突，哈希表会退化成链表，复杂度变为 O(n)。
```

---
